# Umpire Coder

Chrome extension for hockey match official (umpire) performance analysis.
Provides a floating tagging overlay on any live or recorded stream, controls OBS recording via WebSocket, and saves a timestamped JSON event log.

## Phase 1 — Load and Test the Extension

### 1. Generate icons (if missing)

```bash
node extension/scripts/create-icons.js
```

Or use the PowerShell one-liner (no Node required):

```powershell
Add-Type -AssemblyName System.Drawing
foreach ($size in @(16, 48, 128)) {
    $bmp = New-Object System.Drawing.Bitmap($size, $size)
    $g   = [System.Drawing.Graphics]::FromImage($bmp)
    $g.FillRectangle([System.Drawing.Brushes]::Blue, 0, 0, $size, $size)
    $g.Dispose()
    $bmp.Save("extension\icons\icon${size}.png", [System.Drawing.Imaging.ImageFormat]::Png)
    $bmp.Dispose()
}
```

### 2. Load in Chrome

1. Open `chrome://extensions`
2. Enable **Developer mode** (top-right toggle)
3. Click **Load unpacked** → select the `extension/` folder
4. The **Umpire Coder** icon appears in the toolbar

### 3. Test without OBS

All OBS WebSocket calls are **stubbed** in [`obs/websocket.js`](extension/obs/websocket.js).
Every call logs to the background service worker console and resolves immediately, so the full UI
is testable without OBS running.

To inspect background console output:
- `chrome://extensions` → Umpire Coder → **Service Worker** → Inspect

## Usage Flow

```
1. Navigate to a stream (YouTube, Twitch, etc.)
2. Click the Umpire Coder icon → fill in match setup → Start Match
   - OBS recording starts (stubbed)
   - Overlay appears as a floating panel on the stream page
3. Click an umpire, click a tag type, (optionally) add notes → Log Event
4. Repeat for each event
5. End Match → event log downloads as JSON → OBS recording stops (stubbed)
```

The overlay is:
- **Draggable** — drag by the title bar
- **Minimisable** — click the − button
- **Settings-accessible** — click ⚙ to open the settings tab without stopping the match

---

## OBS Setup (one time, real OBS)

1. OBS → **Tools → WebSocket Server Settings** → Enable WebSocket server
2. Set a password, note it
3. In extension Settings: enter host (`localhost`), port (`4455`), password
4. Click **Apply OBS Settings** to push video/output config to OBS
5. After this, never open OBS manually again — Start/End Match control it

## Event Log Format

Saved to your Downloads folder as `YYYYMMDD_CompetitionName_events.json`:

```json
{
  "match": {
    "date": "2026-04-27",
    "competition": "National League",
    "venue": "Example Arena",
    "umpire1": "John Smith",
    "umpire2": "Jane Doe"
  },
  "events": [
    {
      "id": 1,
      "timestamp_elapsed": "00:12:34",
      "timestamp_unix": 1745000000,
      "umpire": "John Smith",
      "tag": "Positioning",
      "notes": "Late to the circle"
    }
  ]
}
```

## Phase 2 (not yet built)

- Inputs: event log JSON + OBS output path
- Clip naming: `UMPIRENAME_TAGTYPE_ELAPSED-TIMESTAMP.mp4`
